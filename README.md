PRINCIPES SOLID
Les principes SOLID sont un ensemble de bonnes pratiques de conception orientée objet permetant de produire un code plus maintenable, flexible et evolutif.
- SRP: une classe ne doit avoir qu'une seule responsabilité.
- OCP: une classe  doit etre ouverte à l'extension mais fermée à la modification
- LSP: une classe dérivée doit pouvoir remplacer sa classe mère sans erreur
- ISP: il vaut mieux plusieurs petites interfaces qu'une seule grosse.
- DIP les classes de haut niveau ne doivent pas dependre des calssses de bas niveaux
